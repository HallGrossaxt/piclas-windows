# Installation (Windows port)

> **piclas-win** builds and runs PICLas natively on **Windows 10/11** using the MSYS2 UCRT64
> toolchain and Microsoft MPI — no WSL, Cygwin, or Linux VM. The native executable is named
> **`piclas-win.exe`**. **This page documents the Windows build only.**
>
> For the original **Linux** installation (apt prerequisites, self-compiled MPI/HDF5/PETSc,
> cluster module environments and Linux troubleshooting), see the upstream original:
> <https://piclas.readthedocs.io/en/latest/userguide/installation.html>

## Prerequisites (MSYS2 UCRT64)

| Component | Notes |
|-----------|-------|
| [MSYS2](https://www.msys2.org/) UCRT64 environment | Build shell — not Cygwin, not WSL |
| MinGW-w64 GCC / GFortran | C + Fortran compilers |
| CMake ≥ 3.10 (4.2.x tested) + Ninja | Configure / build |
| HDF5 2.1.0 (**serial**) | MSYS2 package is serial-only |
| LAPACK / OpenBLAS | Dense linear algebra |
| Microsoft MPI (MS-MPI) | `mpiexec.exe` for MPI builds |
| PETSc 3.24.5 *(optional)* | MSYS2 build is **sequential** (MPI=1 only) |
| CUDA Toolkit 12.x/13.x + VS Build Tools *(optional)* | GPU builds only |

> **MPI + PETSc cannot be combined** — the MSYS2 PETSc package is sequential, so PETSc-enabled
> builds run at MPI=1 only.

These third-party libraries are the same as upstream; on Windows they are provided by MSYS2
packages instead of being self-compiled (`LIBS_BUILD_HDF5=OFF`, `LIBS_BUILD_MATH_LIB=OFF` in the
Windows presets).

## Install the build environment

1. Install [MSYS2](https://www.msys2.org/) and open the **UCRT64** shell.
2. Install the toolchain and libraries:

       pacman -S --needed \
         mingw-w64-ucrt-x86_64-gcc \
         mingw-w64-ucrt-x86_64-gcc-fortran \
         mingw-w64-ucrt-x86_64-cmake \
         mingw-w64-ucrt-x86_64-ninja \
         mingw-w64-ucrt-x86_64-hdf5 \
         mingw-w64-ucrt-x86_64-openblas \
         mingw-w64-ucrt-x86_64-msmpi \
         git python

3. Install the **Microsoft MPI runtime** (`msmpisetup.exe`) from Microsoft so that `mpiexec.exe`
   is available (typically `C:\Program Files\Microsoft MPI\Bin\mpiexec.exe`).
4. *(Optional, GPU builds only)* Install the CUDA Toolkit 12.x/13.x and Visual Studio Build Tools.

## Build piclas-win

From the **UCRT64** shell, in the repository root:

    export TEMP=/tmp && export TMP=/tmp        # system TEMP may be read-only
    cmake --preset windows-ucrt64-mpi          # MS-MPI build (maxwell/DSMC)
    cmake --build --preset windows-ucrt64-mpi

The resulting executable is **`build-<preset>/bin/piclas-win.exe`** (plus `libpiclas.dll`). See the
[README](../../../README.md) for the full preset table (serial, MPI, Poisson, DSMC+GPU, etc.). Meshes
are generated by the [hopr-win](https://github.com/HallGrossaxt/hopr-win) pre-processor port.

## Run

    .\bin\piclas-win.exe parameter.ini [DSMC.ini]
    & "C:\Program Files\Microsoft MPI\Bin\mpiexec.exe" -n 4 .\bin\piclas-win.exe parameter.ini [DSMC.ini]

For everything platform-independent (parameter files, physics, features, usage), the upstream
[PICLas User Guide](https://piclas.readthedocs.io/en/latest/) applies unchanged.
