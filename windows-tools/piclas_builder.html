piclas-win 1.0 Build Configurator

Unofficial Windows port of PICLas 4.1.0 — standalone HTML, works in any browser, no server needed
Workflow: Select options → click Download Build Script → double-click the .bat file to build. The script calls C:\msys64\ucrt64\bin\cmake.exe directly from CMD — no bash, no MSYSTEM tricks needed. All compilers (gfortran, mpifort, ninja) are referenced by absolute path inside the cmake preset, so the correct UCRT64 toolchain is always used.

Equation System

maxwell: full electromagnetic (DGSEM) — EM wave / laser / combined DSMC+PIC-EM simulations. Use with RK3, RK4, RK14, DSMC, FP-Flow, BGK-Flow, Radiation.
poisson: electrostatic Poisson solver (HDG) — electrostatic PIC without radiation. Supported: Euler-Explicit, Higuera-Cary, Boris-Leapfrog, Leapfrog, RK3, RK4, RK14. DSMC, FP-Flow, BGK-Flow, Radiation, ED-DVM, Explicit-FV, PLOESMA fail at configure time.
discrete_velocity: Boltzmann discrete-velocity (FV solver) — use with ED-DVM, PLOESMA, or Explicit-FV.
drift_diffusion: drift-diffusion (HDG+FV) — use with Explicit-FV or RK3/RK4/RK14 with combined field+FV.

Time Integration

RK3 / RK4 / RK14: explicit LSERK Runge-Kutta. Works with maxwell (PP_TimeDiscMethod=1/2/6), poisson (HDG, 501/502/506), or drift_diffusion (HDG+FV). maxwell+RK4 is the correct build for combined DSMC+PIC-EM — DSMC collision physics is enabled at runtime via parameter file (e.g. Particles-DSMC-CollisMode=3).
Boris-Leapfrog: symplectic Boris pusher (PP_TimeDiscMethod=508) — requires poisson. Fails with maxwell.
Leapfrog: 2nd-order leapfrog (PP_TimeDiscMethod=509) — requires poisson. Fails with maxwell.
Euler-Explicit: 1st-order explicit Euler for HDG (PP_TimeDiscMethod=500) — requires poisson.
Higuera-Cary: energy-conserving relativistic Boris variant (PP_TimeDiscMethod=507) — requires poisson.
DSMC: pure Direct Simulation Monte Carlo time loop (PP_TimeDiscMethod=4, no EM field solve). Use with maxwell. For combined DSMC+PIC-EM use RK4 + maxwell instead and enable DSMC at runtime.
FP-Flow / BGK-Flow: stochastic particle methods for transitional flow regimes. Require maxwell and PICLAS_PARTICLES=ON (the default).
Radiation: radiative transfer time loop (PP_TimeDiscMethod=600). Requires maxwell.
ED-DVM: explicit discrete-velocity method (PP_TimeDiscMethod=700, FV solver). Requires discrete_velocity equation system.
Explicit-FV: explicit finite-volume time stepping (PP_TimeDiscMethod=701). Requires discrete_velocity or drift_diffusion.
PLOESMA: plasma+DVM coupling with HDG fields + FV particles (PP_TimeDiscMethod=702). Requires discrete_velocity.

Build Type

Release: full optimisation, no debug symbols — use for production runs.
Debug: no optimisation, full symbols — use when debugging crashes.
RelWithDebInfo: optimised + symbols — good for profiling.
Sanitize: address+undefined sanitizer instrumentation (~10× slower runtime). Use to chase memory bugs / UB. Not compatible with MPI on Windows — sanitizer errors fire inside the MPI library itself; build with LIBS_USE_MPI=OFF.
Profile: instrumented for gprof-style profiling.

Features

Build post-processing tools. With POSTI=ON: piclas2vtk (HDF5 → VTK/ParaView converter) and (unless time method is ED-DVM) superB (standalone B-field pre-processor). Building POSTI also compiles USE_SUPER_B=1 into the piclas binary, which is required for any example that reads a background magnetic field (e.g. HEMPT, gyrotron, *variable_B*). Turn POSTI off only for stripped-down runtime-only builds.
Also build the DMD (Dynamic Mode Decomposition) tool. Required for the NIG_maxwell_RK4/CoaxialCable_DMD family of tests. Only effective if POSTI is ON.
Compile the particle module. Turn OFF for pure field-only builds (e.g. NIG_maxwell_RK4, NIG_convtest_maxwell, CHE_maxwell, CHE_poisson_periodic). Required for DSMC, PIC, FP-Flow, BGK-Flow, Radiation, Leapfrog, Boris-Leapfrog, etc.
Read physical constants from the parameter file instead of hard-coded values. Useful for code verification and sensitivity studies. Required by NIG_sanitize — without it the timestep collapses and the test runs forever.
Print shared-memory window sizes at startup. Used in CHE/NIG Debug+CODE_ANALYZE configurations (e.g. CHE_maxwell, CHE_poisson, NIG_drift_diffusion_explicit-FV). Performance cost is negligible.
Build unit tests. Disable for faster builds. Enable when making solver changes to verify correctness.
Enable multi-process parallel builds using Microsoft MPI. Requires mingw-w64-ucrt-x86_64-msmpi installed via pacman. Uses the windows-ucrt64-mpi preset. HDF5 I/O remains serial. Not compatible with PETSc — the MSYS2 PETSc package is sequential-only and will crash at runtime when launched with more than one MPI rank.
Dynamic load balancing across MPI ranks. Default ON when MPI is enabled. Turn OFF only for ED-DVM/PLOESMA builds (these explicitly require it OFF).
Output measured wall time for MPI_WAIT calls per rank. Useful for identifying load-imbalance hot spots.
Link against the PETSc library for advanced iterative linear solvers (e.g. GMRES, CG with preconditioners). Required for tutorials that use the HDG Poisson solver with PETSc backend. Needs mingw-w64-ucrt-x86_64-petsc installed via pacman. Not compatible with MPI — the MSYS2 PETSc package is built without MPI support; enabling both causes a runtime abort ("MPI-uni install") when launched with more than one process.
FV fluxes not calculated for dimension Z (2D simulations). Available only with discrete_velocity + FV-style time methods.

GPU Acceleration (CUDA)

Enable GPU-accelerated particle position push via CUDA.

What is accelerated (v1.0):
• DSMC position push — batch pos += vel × dt for all active particles. Active when DSMC time method is selected and none of: variable time step, rotating reference frame, granular species, surface flux, axisymmetric geometry.
• RK4 / RK14 LSERK stage push — GPU-accelerated per-stage velocity+position update (Carpenter 1994 / Niegemann 2012 coefficients). Active when maxwell + RK4 or maxwell + RK14 is selected and none of: variable time step, rotating reference frame, coupled power output, field ionization.
• Other time methods: GPU build is valid but particle push runs on CPU.

Requirements:
• NVIDIA GPU (Ampere / RTX 30xx or newer recommended, CC ≥ 8.6)
• CUDA Toolkit 12+ — install from developer.nvidia.com; ensure nvcc.exe is on PATH or CUDA_PATH is set
• VS Build Tools 2022 (free) — nvcc on Windows requires cl.exe as host compiler (MinGW g++ unsupported); install from visualstudio.microsoft.com with the Desktop development with C++ workload; CMake finds cl.exe automatically
• Python 3 — used at link time to patch CUDA object files. Any real Python 3 installation works (python.org, Conda, MSYS2). CMake locates it automatically at configure time. The Windows Store python3.exe stub does not work — disable it under Settings → Apps → Advanced App Settings → App Execution Aliases if you have a real Python installed and still see a Store redirect error.
• cudart64_13.dll must be bundled with the executable for distribution (see §14.7 of the setup guide)

Not compatible with PETSc. GPU + MPI: known race in the per-rank CUDA-context init during the first load-balance redistribution, which can crash the run at exit-code 3 on small / split-mesh examples (Bug-G/CUDA-context race). For CVWM-style examples disable load balance or limit MPI ranks.
CMAKE_CUDA_ARCHITECTURES — GPU compute capability
Match to your GPU generation. A mismatched arch still runs via JIT recompilation but adds a one-time delay at first launch. Requires CUDA Toolkit ≥ 12.8 for CC 120.

Advanced

PICLAS_NODETYPE
Spectral element node distribution. GAUSS: full Legendre-Gauss (interior only). GAUSS-LOBATTO: Legendre-Gauss-Lobatto (includes endpoints). Almost all PICLas tests use GAUSS; GAUSS-LOBATTO is incompatible with poisson and DSMC.
Enable performance counters and timing instrumentation. Adds small overhead; useful for benchmarking hot spots.
Enable runtime sanity checks and assertions. Significant performance hit — only for development/debugging. Required by several CHE/NIG suites (e.g. CHE_poisson, CHE_PIC_maxwell_RK4_p_adaption, NIG_code_analyze_RK4).
PICLAS_INSTRUCTION — target CPU instruction set
-march=x86-64-v2: AVX+SSE4.2, compatible with most CPUs since ~2012. Safe default.
-march=native: maximum performance for the build machine only (binary not portable).
-march=x86-64-v3: AVX2 for Haswell 2013+ CPUs.

      Why a .bat file? Browsers run in a security sandbox and cannot execute programs directly.
      The downloaded script opens a CMD terminal where you can watch the full build output.


      Why not bash? Previous versions launched bash --login and set MSYSTEM=UCRT64, but that is fragile — the MPI compiler wrappers sometimes weren't picked up. The current approach calls C:\msys64\ucrt64\bin\cmake.exe directly from CMD. Because the cmake preset hardcodes every compiler path (gfortran, mpifort, ninja), no shell environment setup is required.
