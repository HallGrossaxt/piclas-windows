piclas-win 0.9 Build Configurator

Unofficial Windows port of PICLas 4.1.0 — standalone HTML, works in any browser, no server needed
Workflow: Select options → click Download Build Script → double-click the .bat file to build. The script calls C:\msys64\ucrt64\bin\cmake.exe directly from CMD — no bash, no MSYSTEM tricks needed. All compilers (gfortran, mpifort, ninja) are referenced by absolute path inside the cmake preset, so the correct UCRT64 toolchain is always used.

Equation System

maxwell: full electromagnetic (DGSEM) — EM wave / laser simulations.
poisson: electrostatic Poisson solver (HDGSEM) — PIC plasma without radiation.
electrostatic_pois: PIC with Poisson, no magnetic self-field.
hdg: Hybridized Discontinuous Galerkin Poisson solver.

Time Integration

RK3/RK4/RK14: explicit Runge-Kutta — general purpose.
Boris-Leapfrog: symplectic, ideal for PIC particle tracking.
Leapfrog: 2nd-order, lightweight — recommended for electrostatic PIC.
RK4-implicit: for stiff systems with disparate time scales.
DSMC: Direct Simulation Monte Carlo — rarefied gas / particle collisions. Use with maxwell. Required for the dsmc-cone-3D tutorial.
FP-Flow / BGK-Flow: stochastic particle methods for transitional flow regimes. Require particles (PICLAS_PARTICLES=ON, the default).

Build Type

Release: full optimisation, no debug symbols — use for production runs.
Debug: no optimisation, full symbols — use when debugging crashes.
RelWithDebInfo: optimised + symbols — good for profiling.

Features

Build post-processing tools (piclas2vtk) for converting HDF5 output to VTK/ParaView format.
Read physical constants from the parameter file instead of hard-coded values. Useful for code verification and sensitivity studies.
Build unit tests. Disable for faster builds. Enable when making solver changes to verify correctness.
Enable multi-process parallel builds using Microsoft MPI. Requires mingw-w64-ucrt-x86_64-msmpi installed via pacman. Uses the windows-ucrt64-mpi preset. HDF5 I/O remains serial. Not compatible with PETSc — the MSYS2 PETSc package is sequential-only and will crash at runtime when launched with more than one MPI rank.
Link against the PETSc library for advanced iterative linear solvers (e.g. GMRES, CG with preconditioners). Required for tutorials that use the HDG Poisson solver with PETSc backend. Needs mingw-w64-ucrt-x86_64-petsc installed via pacman. Not compatible with MPI — the MSYS2 PETSc package is built without MPI support; enabling both causes a runtime abort ("MPI-uni install") when launched with more than one process.

GPU Acceleration (CUDA)

Enable GPU-accelerated particle kernels via CUDA. Requires:
• NVIDIA GPU (Turing/RTX 20xx or newer recommended)
• CUDA Toolkit 12+ from developer.nvidia.com with nvcc on PATH
• VS Build Tools 2022 (free) from visualstudio.microsoft.com — nvcc on Windows requires cl.exe as host compiler; MinGW g++ is not supported by nvcc.
Compatible with MPI. Not compatible with PETSc.
CMAKE_CUDA_ARCHITECTURES — GPU compute capability
Match this to your GPU generation. Wrong arch still runs but recompiles at first launch (slower startup).

Advanced

Enable performance counters and timing instrumentation. Adds small overhead; useful for benchmarking hot spots.
Enable runtime sanity checks and assertions. Significant performance hit — only for development/debugging.
PICLAS_INSTRUCTION — target CPU instruction set
-march=x86-64-v2: AVX+SSE4.2, compatible with most CPUs since ~2012. Safe default.
-march=native: maximum performance for the build machine only (binary not portable).
-march=x86-64-v3: AVX2 for Haswell 2013+ CPUs.
Build directory — relative to C:/Data/PRJ/Piclas/

      Why a .bat file? Browsers run in a security sandbox and cannot execute programs directly.
      The downloaded script opens a CMD terminal where you can watch the full build output.


      Why not bash? Previous versions launched bash --login and set MSYSTEM=UCRT64, but that is fragile — the MPI compiler wrappers sometimes weren't picked up. The current approach calls C:\msys64\ucrt64\bin\cmake.exe directly from CMD. Because the cmake preset hardcodes every compiler path (gfortran, mpifort, ninja), no shell environment setup is required.

      ⚠ Required before running MPI simulations:

      Set HDF5_USE_FILE_LOCKING=FALSE before calling mpiexec on Windows,

        export HDF5_USE_FILE_LOCKING=FALSE

        export PATH=/c/msys64/ucrt64/bin:$PATH
